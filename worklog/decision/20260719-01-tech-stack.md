# decision — 기술스택 / 역할 분리

## 무엇을
- 언어: **Python** (환경에 3.13/3.14 존재)
- 라이브러리: `requests`(디스코드 전송), `PyYAML`(설정) — 둘 다 설치 확인됨
- **역할 분리**: 조사 = Claude(WebSearch/WebFetch/브라우저), 전송·기록 = Python 스크립트

## 왜
- 임의의 채용 사이트를 스크립트로 파싱하면 사이트 구조 변경·봇 차단에 취약.
  → 유연한 조사는 Claude가, 결정적인 전송/중복제거/기록은 스크립트가 맡는 게 안정적.
- 설정은 사람이 편집하므로 주석 가능한 YAML 채택.

## 대안
- 전 과정 순수 스크립트 크롤러 → 사이트별 파서 유지보수 부담이 커서 제외.
- Node.js → 가능하나 Python으로 충분하고 표준 라이브러리 위주로 갈 수 있어 선택.
