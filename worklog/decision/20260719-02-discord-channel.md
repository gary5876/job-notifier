# decision — 전송 채널: 디스코드 웹훅

## 무엇을
알림 전송 채널로 **디스코드 Incoming Webhook** 채택.
웹훅 URL은 `config/secrets.yaml`에 저장 (민감 정보, .gitignore 처리).

## 왜
- 웹훅 URL 하나로 별도 OAuth 없이 즉시 자동 전송 가능.
- 카카오톡("나에게 보내기")은 디벨로퍼 앱 등록 + OAuth 토큰 발급이 필요해 설정 부담이 큼.
- 사용자가 디스코드 사용 확정 + 웹훅 URL 제공함.

## 대안
- Slack: 디스코드와 난이도 비슷. 사용자가 디스코드 선택.
- 카카오톡: 설정 복잡으로 보류.

## 형식
- 공고 1건 = 임베드 1개. 트랙별 색상(사이트=파랑, 회사=초록).
- 메시지당 임베드 10개 제한 → 배치 전송. 실행당 `max_per_run`(기본 15)로 도배 방지.
