# worklog — 작업기록

이 폴더는 **구직 알리미 프로그램 개발 기록**이다. 코드가 아니라 *왜 이렇게 만들었는지* —
결정사항, 진행범위, 논의 내용을 남긴다.

## 구조 (분류별 하위 폴더, 영문만)

```
worklog/
├── decision/   설계·기술 선택 등 확정한 것 (되돌리려면 근거가 필요한 것)
├── progress/   그날그날 무엇을 어디까지 했는지
├── scope/      만들 것 / 안 만들 것의 경계
└── issue/      문제·미결·나중에 볼 것
```

각 폴더 안 파일명: `YYYYMMDD-NN-slug.md` (영문만)
- **YYYYMMDD**: 날짜  ·  **NN**: 그 날짜의 순번 `01`, `02` …  ·  **slug**: 제목(영문 하이픈)

예) `worklog/decision/20260719-02-discord-channel.md`

## 규칙
- 파일·폴더 이름은 **영문만** (한글 금지). 내용은 한글 OK.
- 한 파일 = 한 사안. 짧아도 됨.
- 결정은 *무엇을/왜/대안* 을 남긴다 (나중의 나를 위해).
